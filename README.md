## :eyes: Usage

### Libraries

> Applications grow in size and complexity and in a large organization there are many ways to reuse code to maintain consistency and to lower the development effort. Developing in a modular way can be done using libraries, which are simply a collection of related files that perform a certain task. These are composed together to make up an application.

#### Types of libraries

There are many different types of libraries in a workspace. In order to maintain a certain sense of order, we are using only the below four (4) types of libraries:

- **Feature libraries:** Developers should consider feature libraries as libraries that implement smart UI (with injected services) for Business Use Cases specific to that feature.<br/>

- **UI libraries:** A UI library contains only presentational components.<br/>

- **Data-access libraries:** A data-access library contains services and utilities for interacting with a
back-end system.<br/>

- **Utility libraries:** A utility library contains common utilities and services used by many libraries and applications. It also includes all the code related to State management.<br/>

#### Naming conventions and grouping libraries

Libraries require classifiers to describe their contents and intended purpose. These classifiers help to organize the libraries and to provide a way to locate them.

- **Scope:** Relates to a logical grouping, business use-case or domain. They contain libraries that manage a sub-domain of application logic. We will be using a **_folder structure to denote scope._**<br/>

- **Type:** Relates to the contents of the library and indicates its purpose and usage. **_We will be using prefixes and tags to denote type._** These types are limited to the four (feature-, ui-, data-access- or util-) types as seen in the section above.<br/>

- **Platform:** There can be other classifiers used to differentiate between similar libraries. Platform is one such classifier.<br/>

So we will end up with the following structure: **[scope] / [type]--[platform]**.<br/>
> **example:** releases / data-acces--gateway
